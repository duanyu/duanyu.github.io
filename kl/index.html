KL 散度在 LLM 微调中的直观展示

基本概念

假设 Base Model (P) 对某个问题的回答有两个同样高概率的选项（双峰分布）。Finetuned Model (Q) 如何去拟合它？

Forward KL: D_KL(P || Q)

博学但平庸 (Mean-Seeking)

SFT (监督微调) 的逻辑

Q 会试图覆盖 P 中所有的非零区域。为了不漏掉任何一个正确答案，Q 选择了“既要又要”，结果分布变得非常宽。内容全面但不够犀利。

Toy Example: 词汇预测
Prompt: "长风破浪会有时，直挂云帆___"
Base (P): [济沧海: 0.5, 济深海: 0.4, ...]
FT (Q) 输出: "济沧海" (0.45), "济深海" (0.4)
结果：模型学会了数据集里所有的变体，但概率被平摊了。
Reverse KL: D_KL(Q || P)

精准但保守 (Mode-Seeking)

RLHF (强化学习) 的逻辑

Q 只在 P 概率最高的地方分配概率。如果 Q 尝试覆盖 P 没有的地方，惩罚会极大。结果 Q 锁定了一个“最稳”的答案，放弃了多样性。

Toy Example: 风格对齐
Prompt: "你好"
Base (P): [你好: 0.3, 您好: 0.3, 嘿: 0.1...]
FT (Q) 输出: "您好" (0.95)
结果：模型发现“您好”最安全且奖励高，于是极度收敛到这一个点。

总结对比

特性Forward KL (SFT)Reverse KL (RLHF)
行为Zero-avoiding (不能漏掉 P 的可能性)Zero-forcing (严禁踩到 P 的低频区)
直观感受覆盖所有模式，分布变宽锁定单一模式，分布变窄（坍缩）
LLM 风险容易产生“幻觉”，采样时容易出乱码容易产生“回答复读”或多样性缺失
